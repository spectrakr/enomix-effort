# eNomix 공수 관리 시스템 사용 가이드

## 📌 시스템 개요
Jira의 완료된 Epic 데이터를 자동으로 수집하고, AI 기반 질의응답을 통해 과거 프로젝트 공수 정보를 조회할 수 있는 시스템입니다.

## 🌐 접속 방법
- **웹 페이지**: http://211.63.24.116:9000
- **슬랙 봇**: 슬랙에서 `@enomix-effort` 멘션

---

## 📊 주요 기능

### 1. **공수 데이터 관리** (메인 화면)
- **용도**: 수집된 공수 데이터 조회 및 관리
- **경로**: 메인 화면 우측 "공수 데이터 관리" 탭

#### 기능
- **목록 조회**: 페이지별로 공수 데이터 확인 (20개씩)
- **검색**: Jira 티켓 번호, 제목, Epic 키로 검색
- **카테고리 수정**: 각 작업의 분류(분석/설계/구현 등) 변경 가능
- **삭제**: 잘못된 데이터는 X 버튼으로 삭제 (현재 페이지 유지됨)

---

### 2. **Epic 데이터 동기화**

#### 2-1. Epic 하위 작업 일괄 가져오기 (수동)
- **용도**: 특정 Epic의 하위 작업을 수동으로 수집
- **사용법**:
  1. Epic 키 입력 (예: `ENOMIX-1234`)
  2. "Epic 하위 작업 가져오기" 버튼 클릭
  3. 완료 메시지 확인

- **언제 사용?**
  - 특정 Epic만 업데이트하고 싶을 때
  - WORK 프로젝트 데이터를 수동으로 추가할 때

#### 2-2. 완료된 Epic 자동 동기화
- **용도**: ENOMIX 프로젝트의 완료된 Epic을 자동으로 검색하여 일괄 수집
- **사용법**:
  1. "완료된 Epic 자동 동기화" 버튼 클릭
  2. 백그라운드에서 처리 (시간이 걸릴 수 있음)
  3. 하단 "스케줄러 이력"에서 진행 상황 확인

- **주의사항**:
  - ENOMIX 프로젝트만 자동 검색됩니다
  - WORK 프로젝트는 수동으로 추가해야 합니다

#### 스케줄러 이력
- **위치**: 화면 하단
- **정보**: 동기화 시작/완료 시간, 상태, 추가/업데이트/실패 건수
- **자동 갱신**: 10초마다 자동으로 최신 이력 표시

---

### 3. **AI 질의응답** (메인 화면 좌측)

#### 웹 채팅
- **용도**: 과거 프로젝트 공수 정보 질의
- **사용법**:
  1. 질문 입력란에 궁금한 내용 작성
  2. "전송" 버튼 클릭 (또는 Enter)
  3. AI 답변 확인

#### 질문 예시
```
✅ 추천 질문 형식:
- "ENOMIX-1234 프로젝트 총 공수는?"
- "경남은행 프로젝트 정보 알려줘"
- "삼성카드 프로젝트 얼마나 걸렸어?"

❌ 비추천:
- "저번에 했던 프로젝트" (Epic 키나 고객사명 필요)
- "빨리 끝난 프로젝트" (구체적인 정보 필요)
```

#### Epic 정보 조회 시 제공되는 정보
1. **기본 정보**: Epic 키, 제목, 총 공수
2. **단계별 공수**: 세팅, 분석/설계, 구현, 테스트, 반영/이행, 기타
3. **담당자별 공수**: 2명 이상 투입 시 담당자별 공수 표시
4. **프로젝트 요약**: AI가 자동 생성한 프로젝트 개요
5. **고객사 특성 분석** (있는 경우):
   - 난이도 등급 (11~44, 낮을수록 협조적)
   - 요구사항 명확성, 개발 유연성, 고객 소통 지수
   - 버퍼 필요 여부 또는 표준 공수 충분 여부
   - 참고자료 링크 (Google Sheets)
6. **주요 작업 목록**: 상위 10개 작업 상세

## ⚠️ 주의사항

### 1. WORK 프로젝트 데이터
- 자동 동기화 대상이 아닙니다
- 필요시 "Epic 하위 작업 일괄 가져오기"로 수동 추가

### 2. 데이터 삭제
- 목록에서 X 버튼으로 삭제 가능
- **복구 불가**: 삭제 전 신중하게 확인

### 3. 카테고리 수정
- Epic 동기화 시에도 기존 카테고리 정보는 유지됩니다
- 수동으로 분류한 카테고리는 덮어씌워지지 않습니다

### 4. 고객사 난이도 정보
- `data/docs/customer_weights.json` 파일 기반
- 새 고객사 추가 시 해당 파일 업데이트 필요

## 🐛 문제 해결

### 동기화가 너무 느림
- "완료된 Epic 자동 동기화"는 시간이 걸릴 수 있습니다 (수 분~수십 분)
- 스케줄러 이력에서 진행 중인지 확인
- 특정 Epic만 필요하면 "수동 가져오기" 사용

### Epic 정보가 조회되지 않음
- Epic 키가 정확한지 확인 (예: `ENOMIX-1234`)
- 해당 Epic이 시스템에 수집되었는지 "공수 데이터 관리"에서 확인
- 수집 안 되었다면 "Epic 하위 작업 가져오기"로 추가

### 질의응답이 이상함
- 구체적인 Epic 키나 고객사명을 질문에 포함
- "ENOMIX-1234 총 공수"처럼 명확하게 질문

### 웹 페이지가 응답하지 않음
- 페이지 새로고침 시도 (F5 또는 Ctrl+R)
- 브라우저 캐시 삭제 후 재접속
- 다른 브라우저에서 접속 시도
- 계속 문제 발생 시 시스템 관리자에게 문의
