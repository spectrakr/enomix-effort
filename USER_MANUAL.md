## 시스템 소개

### 공수산정기란?

공수산정기는 과거 Jira 티켓 데이터를 기반으로 새로운 개발 작업의 공수를 자동으로 예측해주는 AI 기반 시스템입니다.

### 주요 기능

- **자연어 질문**: "로그인 기능 개발 공수는?" 같은 자연스러운 질문으로 공수 조회
- **과거 데이터 기반**: 실제 완료된 프로젝트 데이터를 바탕으로 정확한 예측
- **다양한 접근 방법**: 웹 또는 슬랙 어디서든 사용 가능
- **피드백 학습**: 사용자 피드백을 통해 지속적으로 답변 품질 향상

### 시스템 구성

- **웹 인터페이스**: 브라우저에서 질문하고 답변 확인
- **슬랙 봇**: 슬랙에서 바로 질문하고 답변 받기
- **AI 엔진**: GPT-4o-mini + RAG 기술로 정확한 답변 생성

## 웹 인터페이스 사용법

### 접속 방법

- 브라우저에서 `http://211.63.24.116:9010/effort-management/effort-management.html` 접속

### 메인 화면 구성

```
┌─────────────────────────────────────┐
│  공수 산정 AI 어시스턴트             │
├─────────────────────────────────────┤
│  [질문 입력창]                       │
│  ▼ 질문을 입력하세요...              │
│                          [전송 버튼] │
├─────────────────────────────────────┤
│  대화 내역                           │
│  ┌─ 사용자: 로그인 기능 공수?       │
│  └─ AI: Jira 티켓: ENOMIX-123...   │
│     👍 👎 (피드백 버튼)              │
└─────────────────────────────────────┘
```

### 질문하기

#### 1. 질문 입력

```
좋은 질문 예시:
✅ "로그인 기능 개발 공수는?"
✅ "사용자 관리 시스템 Story Points는?"
✅ "API 연동 작업 기간이 얼마나 걸리나요?"
✅ "채널추가 공수"
✅ "이력 조회 API"

피할 질문 예시:
❌ "안녕하세요"
❌ "날씨가 어때요?"
❌ "123" (숫자만)
```

#### 2. 답변 확인

시스템은 다음 정보를 제공합니다:

```markdown
📊 공수 산정 답변

Jira 티켓: ENOMIX-123
제목: 로그인 기능 개발
Story Points: 5
예상공수: 5일
산정이유: 기본 로그인 + OAuth 연동 포함
담당자: 홍길동
개발 요구사항: 사용자 인증 및 세션 관리...

참조 문서:
- [ENOMIX-123] 로그인 기능 개발
- [ENOMIX-456] 인증 시스템 구축
```

#### 3. 여러 프로젝트 비교

동일한 기능이 여러 프로젝트에서 개발된 경우 범위로 제시됩니다:

```markdown
예상공수: 3~5일

이전 프로젝트 사례:
- A 프로젝트: 3일 (기본 기능만)
- B 프로젝트: 5일 (OAuth + 2FA 포함)
- C 프로젝트: 4일 (표준 구현)
```

### 답변이 없는 경우

```markdown
❌ 해당 기능에 대한 공수 산정 데이터가 현재 등록되어 있지 않습니다.

다음과 같이 시도해보세요:
1. 질문을 더 구체적으로 작성 (예: "로그인" → "로그인 기능 개발")
2. 유사한 키워드로 재질문 (예: "등록" → "추가")
3. 운영자에게 데이터 추가 요청
```

## 슬랙 봇 사용법

### 봇 호출 방법

#### 1. 채널에서 멘션

```
@공수산정봇 로그인 기능 개발 공수는?
```

#### 2. DM (다이렉트 메시지)

```
1. 슬랙 사이드바에서 "공수산정봇" 클릭
2. 메시지 입력: "채널추가 공수"
3. 답변 확인
```

### 슬랙 봇 명령어

#### 도움말

```
@공수산정봇 도움말
또는
help
```

**응답:**
```
🤖 공수 산정 봇 도움말

사용 가능한 명령어:
• `공수 산정` - 공수 산정 관련 질문
• `통계` - 공수 산정 통계 조회
• `도움말` - 이 도움말 표시

예시 질문:
• "로그인 기능 개발에 얼마나 걸릴까?"
• "사용자 관리 시스템 공수는?"
• "API 개발 시간이 얼마나 필요해?"
```

#### 통계 조회

```
@공수산정봇 통계
또는
stats
```

**응답:**
```
📊 공수 산정 통계

• 총 데이터 수: 1,234개
• 총 Story Points: 5,678점
• 평균 Story Points: 4.6점
```

## 피드백 제공하기

### 피드백의 중요성

사용자 피드백은 시스템이 더 정확한 답변을 제공하도록 학습하는 데 사용됩니다.

### 웹에서 피드백 제공

#### 긍정 피드백 (👍)

답변이 정확하고 도움이 되었을 때 클릭합니다.

```
[AI 답변]
Jira 티켓: ENOMIX-123
예상공수: 5일
...
[👍] [👎]  ← 여기서 👍 클릭
```

**효과:**
- 동일한 질문에 대해 더 빠른 답변 제공
- 시스템이 이 답변을 우선적으로 사용

#### 부정 피드백 (👎)

답변이 부정확하거나 관련 없을 때 클릭합니다.

```
[AI 답변]
Jira 티켓: ENOMIX-999 (관련 없음)
...
[👍] [👎]  ← 여기서 👎 클릭
```

**효과:**
- 시스템이 다른 답변을 검색
- 이 답변은 더 이상 추천되지 않음

### 슬랙에서 피드백 제공

#### 이모지 리액션으로 피드백

```
[봇 응답]
📊 공수 산정 답변
...

👍 도움이 되었나요? 👎
```

**사용 방법:**
1. 봇 메시지에 마우스 오버
2. 이모지 추가 클릭
3. 👍 (thumbsup) 또는 👎 (thumbsdown) 선택

**지원하는 이모지:**
- 긍정: 👍 `:+1:`, `:thumbsup:`, `:white_check_mark:`
- 부정: 👎 `:-1:`, `:thumbsdown:`, `:x:`

#### 피드백 확인 메시지

```
[봇 자동 응답]
👍 피드백이 저장되었습니다. 감사합니다!
```

또는 (중복 피드백 시)

```
👍 피드백이 반영되었습니다. (총 3회) 감사합니다!
```

### 피드백 변경하기

#### 실수로 잘못 누른 경우

**웹:**
1. 반대 버튼 클릭 (👍 → 👎 또는 👎 → 👍)
2. 자동으로 피드백 타입 변경됨

**슬랙:**
1. 기존 이모지 제거
2. 올바른 이모지 추가
3. 마지막 이모지가 최종 피드백으로 저장

## 자주 묻는 질문

### Q1. 답변이 너무 오래 걸려요

**A:** 
- 첫 질문은 5-10초 소요됩니다 (정상)
- 긍정 피드백된 질문은 1-2초로 빨라집니다
- 네트워크 상태를 확인해주세요

### Q2. 답변이 정확하지 않아요

**A:**
1. 부정 피드백(👎) 제공
2. 질문을 더 구체적으로 수정
3. 유사한 키워드로 재질문
4. 운영자에게 데이터 확인 요청

### Q3. 슬랙 봇이 응답하지 않아요

**A:**
1. 봇을 멘션했는지 확인 (`@공수산정봇`)
2. DM인 경우 멘션 없이 바로 질문
3. 서버 상태 확인 (운영자 문의)
4. 다시 시도

### Q4. 어떤 질문을 할 수 있나요?

**A:**
공수 산정과 관련된 모든 질문 가능합니다:
- 기능 개발 공수
- API 작업 시간
- 시스템 구축 기간
- Story Points 조회
- 유사 기능 비교

### Q5. 여러 프로젝트 비교가 가능한가요?

**A:**
네, 자동으로 비교됩니다:
```
예상공수: 3~5일

- 최소: 3일 (기본 기능)
- 최대: 5일 (확장 기능 포함)
- 평균: 4일
```

### Q6. 띄어쓰기가 중요한가요?

**A:**
아니요, 시스템이 자동으로 처리합니다:
- "이력 조회 API" = "이력조회 API" = "이력조회API"

### Q7. 피드백은 언제 제공하나요?

**A:**
항상 제공 가능합니다:
- 답변이 정확할 때: 👍
- 답변이 부정확할 때: 👎
- 처음엔 정확했는데 나중에 부정확: 👍 → 👎 변경

### Q8. 데이터는 얼마나 자주 업데이트되나요?

**A:**
- 운영자가 Jira 데이터를 주기적으로 업데이트
- 피드백은 실시간 반영
- 새로운 데이터는 즉시 검색 가능

### Q9. 개인정보는 안전한가요?

**A:**
- 질문과 답변만 저장됩니다
- 피드백 시 사용자 ID만 기록 (익명화)
- 슬랙 사용자명은 통계 목적으로만 사용

### Q10. 여러 명이 동시에 사용 가능한가요?

**A:**
네, 제한 없이 동시 사용 가능합니다:
- 웹과 슬랙 동시 사용 가능
- 각 사용자별 독립적인 세션
